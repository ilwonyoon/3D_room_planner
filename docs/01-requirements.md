## 1. Product Vision

> **"누구나 자기만의 인테리어를 쉽게 실현할 수 있는 AI Floor Planner"**

- 오늘의집 미션 "이렇게 살아보고 싶다는 꿈을 현실로 만든다"의 XR 확장
- 사용자의 **공간 제약 안에서** 인테리어를 상상·실현할 수 있도록 돕는 것
- 핵심 문제 = **"공간문제"**
  - 좁은 공간 / 바꿀 수 없는 가구 / 동선 / 공간적 상상력 부족

## 2. 타깃 유저 (PO User)

**"도로 한복판에 놓인 인테리어 아기 유저"**
- 지금 당장 공간 기획이 필요한 상황에 던져진 사람
- 엄청난 인테리어 하려는 건 아니지만 이왕이면 적당히 마음에 드는 방을 꾸며 보람을 느끼고 싶음
- 스스로의 인테리어 자신감이 없어 **무엇부터 정해야 할지 모름**

### 유저 Pain Points (Insight)
1. **내 공간에서 상상되지 않아 고통스럽다** (다른 집 레퍼런스는 내 공간과 달라서 무의미)
2. **모든 선택지가 열려있는데 판단 기준이 없어서 고통스럽다**

→ 질문과답변 게시판에 올려도 아무도 답해주지 않음.

## 3. Solution Directions (3축)

| Dir | 이름 | 본질 | MVP 연결 |
|-----|------|------|---------|
| **D1** | 유저사이드 코스트 낮추며 실제 공간 구현 | 유저가 자기 방을 손쉽게 재현 (도면 불러오기 / RoomPlan 캡처 / 그림 그리기) | **공간 입력 UX** |
| **D2** | 상상 코스트 낮추기 | low-fi 렌더링 / Phone-based MR / hi-fi / HMD | **3D 뷰어 피델리티** — 프로토타입은 "포켓룸 수준 low-fi" 타겟 |
| **D3** | 개인화 인테리어 제안 (Key Leverage) | LLM 기반 목적/용도/취향/예산 → 최적 배치 | **AI 배치 + 대화형 편집** |

## 4. 최종 솔루션 시나리오 (재현 목표)

Deck p.12 "최종 솔루션 예시":
1. 인테리어 고민 유저 진입
2. 자신의 공간을 **간단히 입력** → 내 방이 화면에 생성
3. "예산은 300만원인데, 자는 공간과 공부하는 공간을 분리하고 싶어" 니즈 입력
4. 니즈에 맞는 내 방의 모습 출력
5. AI와 **대화하며 디테일 다듬기** ("좀 더 모던한 스타일로")
6. 사용된 **예산**과 **구매 가능한 상품** 확인
7. **내 방에 실현** → 인테리어 고민 해결

추가 시나리오 (p.21):
- 이사 시 도면 입력 → 기존 방 가구 재배치
- "아이 방을 꾸며줘. 나이 5살, 2층 침대" → AI 추천
- 조명 시뮬레이션, 무몰딩 시공 프리뷰
- 스타일링샷 이미지 → 3D로 구현 + 실제 가구 매칭
- 퍼소나 기반 배치 추천
- 내 방에 물건 놓으면 연속 추천
- 고양이 입양 대비 시뮬
- 크리에이터가 구성한 패키지 판매

## 5. 모바일 프로토타입 MVP 범위 (재현 우선)

오리지널 프로토타입 재현 + 2026 기술로 업그레이드를 목표로, **초기 스코프**:

### 5.1 MUST (프로토타입 v1)
- [ ] **375×812 모바일 웹** 뷰포트 (iPhone X ~ 14 표준)
- [ ] **포켓룸 스타일 3D 렌더**: 한 면이 잘린 아이소메트릭 박스룸, 따뜻한 조명
- [ ] **방 공간 입력**: 가로/세로/높이 숫자 입력 → 실시간 박스룸 생성
- [ ] **가구 라이브러리**: CC0/free GLB 30~50 아이템 (침대, 소파, 책상, 의자, 수납, 조명, 식물, 가전)
- [ ] **드래그 배치**: 터치로 가구 선택 → 바닥 위 drop → 회전/이동/삭제
- [ ] **벽 스냅**: 벽 근처에서 자동 정렬
- [ ] **카메라 컨트롤**: 원터치 회전(궤도) / 핀치 줌 / 한 손가락 팬 (모바일 한정)
- [ ] **선택 피드백**: 탭하면 하이라이트 + 삭제/복제 액션바
- [ ] **저장/로드**: localStorage → 방 상태 JSON

### 5.2 SHOULD (v1.1 ~ AI 통합)
- [ ] **AI 배치 제안 버튼**: 빈 방 → 사용자 텍스트 ("예산 300, 1인 원룸, 따뜻한 무드") → LLM이 furniture placement JSON 반환 → 자동 배치
- [ ] **대화형 편집**: 하단 chat bar → "좀 더 모던하게" → 기존 배치 수정
- [ ] **예산 집계**: 사용된 가구의 가격 합 (mock 데이터)
- [ ] **공유/저장**: 캡쳐 이미지 내보내기

### 5.3 NICE-TO-HAVE (v2)
- [ ] 조명 모드 토글 (주간/저녁/밤)
- [ ] 2D 평면도 이미지 업로드 → walls 자동 추출
- [ ] iOS Safari 연동 RoomPlan (USDZ) 업로드
- [ ] 스타일링샷(참조 이미지) → 3D 재현

## 6. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| 초기 로드 | 3G 연결 가정, TTFB+Hero 뷰 < 5초, JS gzip < 400KB 초기 |
| 3D 프레임레이트 | 중급 모바일(iPhone 12, Galaxy A54급)에서 **60fps 유지**, 최저 45fps |
| 인터랙션 지연 | 터치 → 가구 선택 반응 < 100ms |
| 메모리 | WebGL 메모리 < 256MB (iOS Safari 한계 고려) |
| 오프라인 | PWA — 한번 로드된 방은 오프라인 편집 가능 |
| 접근성 | Dynamic Type, VoiceOver 가구명, 색약 대비 셀렉션 |
| 다국어 | 한국어 우선, 영어 i18n 구조만 |

## 7. 성공 지표 (프로토타입)

- 새 유저가 앱 진입 → **첫 방 생성 완료까지 3분 이내**
- **10개 이상 가구를 배치한 방 비율** ≥ 40%
- AI 배치 버튼 사용률 ≥ 30%
- 60fps 유지율 (perf tracking) ≥ 80%

## 8. 비-목표 (Non-goals) — 스코프 관리

- VR/AR 네이티브 (HMD는 장기 비전, 프로토타입 out)
- 실제 결제/주문 integration
- 크리에이터 플랫폼 (Phase 2의 커뮤니티 기능)
- 정밀 건축 CAD 수준 정확도 (인테리어 느낌 우선)
- 실시간 멀티 유저 편집
